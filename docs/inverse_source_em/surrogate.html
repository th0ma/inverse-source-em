inverse_source_em.surrogate

Surrogate models and unified forward API.

This subpackage provides all surrogate-based forward models used throughout the inverse-source-em framework. These models approximate the analytical PhysicsTM solver and enable fast, differentiable forward evaluations for:

  • dataset generation
  • inverse-model training
  • large-scale experiments where the analytical solver is too slow

Included components

SurrogateMLP Base multilayer perceptron architecture used to approximate the complex-valued TM-mode boundary fields.

SurrogateEM Unified surrogate forward model that loads trained MLPs for Esurf and Hsurf and exposes the same API as the analytical PhysicsTM solver.

SurrogateWrapper Compatibility layer that adapts a SurrogateEM instance to a fully PhysicsTM-compatible interface, including broadcasting and batch utilities.

Public API

The following classes are re-exported at the package level:

  • :class:SurrogateMLP
  • :class:SurrogateEM
  • :class:SurrogateWrapper

Notes

This subpackage is used by all regression pipelines and by the dataset generators that rely on surrogate-based forward evaluations.

 40    """
 41    A configurable multilayer perceptron (MLP) surrogate model.
 42
 43    This network approximates the TM-mode boundary fields produced by the
 44    analytical PhysicsTM solver. It maps normalized geometric features and
 45    observation angles to the real and imaginary parts of the surface field.
 46
 47    Parameters
 48    ----------
 49    input_dim : int, optional
 50        Dimensionality of the input feature vector. Default is 5.
 51        Expected input format:
 52            [rho_norm, cos(phi_s), sin(phi_s), cos(theta), sin(theta)]
 53    output_dim : int, optional
 54        Dimensionality of the output vector. Default is 2.
 55        Output format:
 56            [Re(field), Im(field)]
 57    hidden_dim : int, optional
 58        Width of each hidden layer. Default is 128.
 59    num_layers : int, optional
 60        Number of hidden layers. Default is 4.
 61    activation : {"relu", "gelu", "tanh"}, optional
 62        Activation function used in all hidden layers. Default is "relu".
 63
 64    Notes
 65    -----
 66    - The model is fully differentiable and suitable for training with MSE loss.
 67    - Designed for use inside the surrogate forward pipeline.
 68    - The architecture is intentionally minimal to ensure fast inference.
 69    """

158        """
159        Build normalized input features for the surrogate MLP.
160
161        Parameters
162        ----------
163        rho : array-like
164            Radial coordinate(s) of the source.
165        phi : array-like
166            Angular coordinate(s) of the source.

176        rho = np.asarray(rho)
177        phi = np.asarray(phi)
178        theta = np.asarray(theta)
179
180        rho_norm = rho / self.R
181
182        X = np.column_stack([
183            rho_norm,
184            np.cos(phi), np.sin(phi),
185            np.cos(theta), np.sin(theta)
186        ])
187
188        return torch.from_numpy(X).to(device)

151    def batch_forward(self, rho_t, phi_t, I_t, theta_obs):
152        """
153        Compute full boundary fields for a batch of sources and observation angles.
154
155        This method is used by inverse regression models during training to
156        generate the complete set of boundary fields (E_r, E_i, H_r, H_i) for
157        each sample in the batch.
158
159        Parameters
160        ----------
161        rho_t : torch.Tensor, shape (B,)
162            Radial coordinates of the sources.
163        phi_t : torch.Tensor, shape (B,)
164            Angular coordinates of the sources.
165        I_t : torch.Tensor, shape (B,)
166            Source strengths.
167        theta_obs : ndarray of shape (M,)
168            Observation angles in radians.
169
170        Returns
171        -------
172        E_r, E_i, H_r, H_i : torch.Tensor
173            Real and imaginary parts of the electric and magnetic fields,
174            each of shape (B, M).
175
176        Notes
177        -----
178        - All computations are performed using NumPy, then converted back to torch.
179        - Broadcasting is handled automatically.
180        - This method is optimized for training loops.
181        """
182        rho = rho_t.detach().cpu().numpy()
183        phi = phi_t.detach().cpu().numpy()
184        I   = I_t.detach().cpu().numpy()
185
186        batch = rho.shape[0]
187        M = theta_obs.shape[0]
188
189        # Broadcast to (B, M)
190        rho_arr   = np.repeat(rho[:, None],   M, axis=1)
191        phi_arr   = np.repeat(phi[:, None],   M, axis=1)
192        I_arr     = np.repeat(I[:, None],     M, axis=1)
193        theta_arr = np.repeat(theta_obs[None, :], batch, axis=0)
194
195        # Flatten for batch inference
196        rho_f   = rho_arr.reshape(-1)
197        phi_f   = phi_arr.reshape(-1)
198        I_f     = I_arr.reshape(-1)
199        theta_f = theta_arr.reshape(-1)
200
201        # Surrogate forward
202        E = self.sur.batch_Esurf(rho_f, phi_f, theta_f) * I_f
203        H = self.sur.batch_Hsurf(rho_f, phi_f, theta_f) * I_f
204
205        # Reshape back to (B, M)
206        E = E.reshape(batch, M)
207        H = H.reshape(batch, M)
208
209        device = rho_t.device
210        E_r = torch.from_numpy(E.real).to(device)
211        E_i = torch.from_numpy(E.imag).to(device)
212        H_r = torch.from_numpy(H.real).to(device)
213        H_i = torch.from_numpy(H.imag).to(device)
214
215        return E_r, E_i, H_r, H_i
